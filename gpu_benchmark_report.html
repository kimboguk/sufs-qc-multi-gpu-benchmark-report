🔍 주요 발견 사항

1. GPU 가속 효율 구간

≤10 qubits: CPU 권장
15-30 qubits: Single GPU 최적
31-32 qubits: Multi-GPU 필수
33 qubits: 가능하나 느림

2. Multi-GPU의 가치

양적 개선: 30 qubits에서 1.6배 속도 향상
질적 개선: 31-32 qubits를 실행 가능하게 만듦 (2³ = 8배 더 큰 문제 공간)

3. 메모리 제약

  • 각 큐비트는 상태 공간을 2배로 증가시킴
  • 33 qubits: 128GB (한계선)
  • 34 qubits: 256GB (불가능)
  • 더 큰 시뮬레이션을 위해서는 추가 GPU 또는 클러스터 필요

코드 최적화 팁

25+ qubits 회로에는 항상 Multi-GPU 활성화: 
sim = AerSimulator(method='statevector', device='GPU')
result = sim.run(circuit, shots=1000,
                 blocking_enable=True,      # 4 GPU 분산
                 blocking_qubits=23,         # 각 GPU 청크 크기
                 batched_shots_gpu=True).result()
